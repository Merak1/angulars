HTML

¿Qué significa?

Hyper Text Markup Language

Un lenguaje de marcado o lenguaje de marcas es una forma de codificar un documento que, junto con el texto, incorpora etiquetas o marcas que contienen información adicional acerca de la estructura del texto o su presentación.

¿De qué está formado?

HTML Tags o etiquetas

Comienzan con un caracter <, y terminan con un caracter >. Entre esos caracteres está el nombre de la etiqueta, y quizá algunos atributos dependiendo de la etiqueta. La mayoría de los atributos toman un valor también. Algunos atributos son requeridos, algunos son opcionales

La mayoría de las etiquetas contienen otras etiquetas y así definen la estructura de la página

Tags esenciales

  • html: Define el documento como html. Todas las otras etiquetas deben estar dentro de esta
  • head: Contiene información de la página, pero no será mostrada. Los estilos se definen ahí
  • title: Muestra el título en la pestaña del navegador
  • body: La información visible del documento

Atributos de las etiquetas

Los atributos que se representan dentro de las eqtiquetas se definen principalmente con símbolos de igual y un valor, o en algunos casos, exclusivamente el nombre del atributo

Estilo de las etiquetas

Cada etiqueta tiene por defecto ciertos estilos y pueden ser sobreescritos usando el atributo style="" o una Hoja de estilos en cascada CSS

Propiedades de estilo importantes

Display

  • inline: En línea, cupa sólo el espacio del contenido, no puede aplicarsele ancho o altura
  • block: Empieza en una nueva línea y ocupa todo el ancho posibles
  • inline-block: En línea, pero puede aplicarsele ancho y altura
  • flex
  • grid

Margin, padding y border

Son elementos que definen espacio entre un elemento y otro, o entre su contenido

position

define el metodo que utilizará el elemento para posicionar el elemento

  • static: posicionamiento normal. No es afectado por propiedades: top, bottom, left, rigth
  • relative: Deja el espacio que normalmente ocuparía, pero se acomoda según: t,b,l,r
  • fixed: Abandona el espacio que normalmente ocupa y se posiciona según el "viewport"
  • absolute: Abandona el espacio que normalmente ocupa y se posiciona según su padre con position distinta a static
  • sticky: Se posiciona según la posición del scroll del usuario. Cambia entre relative y fixed
