GIT

Manejador de Versiones

Historia

Por a'i de 2005

Surge un problema. El Kernel de Linux se desarrolla y distribuye utilizando un software "privativo" llamado BitKeeper, cuyos creadores deciden quitarles el acceso gratuito que les había otorgado

El equipo de Linux, y en especial uno de sus creadores, Linus Torvalds, deciden crear su propio software de control de versiones distribuido (DVCS), juntando varias de las características que les habían gustado y renovando las que no

Características

  • Velocidad
  • Simplicidad
  • Desarrollo no lineal
  • Free
  • Capacidad de manejar proyectos grandes

"Posiblemente el avance en desarrollo software más importante de los últimos 10 años"

¿Cómo funciona git?

Para entender el funcionamiento de git hay que entender dos conceptos

  1. Commit: Estado del proyecto en un punto en el tiempo
  2. Branch: Apuntador a un commit

Estado de los archivos

  1. Commited: El archivo ha sido agregado a la base local de git
  2. Modified: El archivo ha sido modificado en comparación a su verión "commiteada"
  3. Staged: El archivo ha sido "marcado" para ser agregado como parte del siguiente commit
